Script pour supprimer les utilisateurs inactifs (RGPD)

Politique de conservation des données

  • Utilisateurs sans billets : Suppression après 1 an d'inactivité
  • Utilisateurs avec billets : Suppression 6 mois après la date du dernier billet si inactif
  • Notification : Email envoyé 1 mois avant toute suppression pour informer l'utilisateur

Script Python

Fonctionnement du script

Ce script est exécuté quotidiennement (1 fois par jour) pour vérifier les comptes et appliquer la politique de conservation des données :

  1. Il récupère tous les utilisateurs (non admin) de la plateforme
  2. Pour chaque utilisateur, il vérifie s'il possède des billets
  3. Les comptes sans billets inactifs depuis plus d'un an sont supprimés
  4. Les comptes avec billets sont supprimés 6 mois après la date du dernier billet
  5. Dans les deux cas, une notification par email est envoyée un mois avant la suppression

Cette implémentation nous permet de respecter le principe de limitation de la conservation du RGPD, en ne conservant pas les données au-delà de ce qui est nécessaire.

Logique des tickets

Modèle de données et gestion des tickets

  • Le modèle Ticket représente l'achat global, associé à un utilisateur et un match
  • Chaque TicketCategory définit une catégorie de billet (Silver, Gold, Platinium) avec un prix et un nombre
  • Le modèle SingleTicket génère des billets individuels avec des identifiants uniques
