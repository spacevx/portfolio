Intégration Continue (CI)

Tests automatisés sur chaque push

Une action GitHub a été configurée pour vérifier automatiquement la qualité du code à chaque push (un test unitaire) :

  • Vérification des types TypeScript
  • Build du projet pour s'assurer qu'il compile correctement

La vérification permet de détecter les erreurs et d'assurer une bonne qualité de code (QoL).

GitHub Workflow Code

name: TypeScript Build and Release

on:
  push:
    branches: [ "main" ]
  pull_request:
    branches: [ "main" ]

# Si on ne possède pas ces permissions il ne voudra pas faire la release
permissions:
  contents: write
  issues: write
  pull-requests: write

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v3
        with:
          fetch-depth: 0

      - name: Setup Node.js
        uses: actions/setup-node@v3
        with:
          node-version: 20.18.0
          cache: 'npm'

      - name: Install Dependencies
        run: npm ci

      - name: TypeScript Type Check
        run: npx tsc --noEmit

      - name: Build Project
        run: npm run build

      - name: Get Previous Version
        id: prev_version
        run: |
          if git rev-parse HEAD^1 >/dev/null 2>&1; then
            git show HEAD^:package.json > previous_package.json || echo '{"version": "0.0.0"}' > previous_package.json
          else
            echo '{"version": "0.0.0"}' > previous_package.json
          fi
          PREV_VERSION=$(node -p "require('./previous_package.json').version")
          echo "version=$PREV_VERSION" >> $GITHUB_OUTPUT

      - name: Get Current Version
        id: current_version
        run: |
          CURRENT_VERSION=$(node -p "require('./package.json').version")
          echo "version=$CURRENT_VERSION" >> $GITHUB_OUTPUT

      - name: Check Version Change
        id: version_check
        run: |
          PREV_VERSION=${{ steps.prev_version.outputs.version }}
          CURRENT_VERSION=${{ steps.current_version.outputs.version }}
          
          IFS='.' read -r PREV_MAJOR PREV_MINOR PREV_PATCH <<< "${PREV_VERSION//./ }"
          IFS='.' read -r CURR_MAJOR CURR_MINOR CURR_PATCH <<< "${CURRENT_VERSION//./ }"
          
          if [ "$PREV_MAJOR" != "$CURR_MAJOR" ] || [ "$PREV_MINOR" != "$CURR_MINOR" ]; then
            echo "create_release=true" >> $GITHUB_OUTPUT
          else
            echo "create_release=false" >> $GITHUB_OUTPUT
          fi

      - name: Create Git Tag
        if: steps.version_check.outputs.create_release == 'true' && github.event_name == 'push'
        run: |
          git config --local user.email "action@github.com"
          git config --local user.name "GitHub Action"
          git tag -a v${{ steps.current_version.outputs.version }} -m "Release version ${{ steps.current_version.outputs.version }}"
          git push origin v${{ steps.current_version.outputs.version }}

      - name: Prepare Release Files
        if: steps.version_check.outputs.create_release == 'true' && github.event_name == 'push'
        run: |
          mkdir -p release_build
          
          cp -r dist/ release_build/
          cp -r src/ release_build/
          
          cp package.json release_build/
          cp tsconfig.json release_build/
          cp README.md release_build/ 2>/dev/null || true
          
          echo "node_modules/" > release_build/.gitignore
          echo ".github/" >> release_build/.gitignore
          echo ".git/" >> release_build/.gitignore
          echo ".idea/" >> release_build/.gitignore
          
          cd release_build
          zip -r ../release.zip . -x "**/.git*" "**/node_modules/*" "**/.idea/*" "**/.github/*"

      - name: Create GitHub Release
        if: steps.version_check.outputs.create_release == 'true' && github.event_name == 'push'
        uses: softprops/action-gh-release@v1
        with:
          files: release.zip
          tag_name: v${{ steps.current_version.outputs.version }}
          name: Release v${{ steps.current_version.outputs.version }}
          body: |
            Release version ${{ steps.current_version.outputs.version }}
            
            Cette release contient :
            - Code source (dossier src/)
            - Build compilé (dossier dist/)
            - Fichiers de configuration (package.json, tsconfig.json)
            
            Pour installer :
            1. Décompressez l'archive
            2. Exécutez `npm install`
            3. Utilisez `npm start` pour lancer l'application
          draft: false
          prerelease: false
          token: ${{ secrets.GITHUB_TOKEN }}

Déploiement Continu (CD)

Releases automatiques basées sur les commits

Le workflow est configuré pour créer automatiquement des releases lors de modifications :

  • Détection du changement de version dans le fichier package.json
  • Création d'un tag Git pour spécifier la version
  • Archive (ZIP) contenant la version compilée (dist)
  • Publication de la release GitHub avec les instructions

Fonctionnement du processus de release

  1. Le système extrait les versions précédente et actuelle du package.json
  2. Si une modification majeure ou mineure est détectée (changement de version x.y.z), une release est créée
